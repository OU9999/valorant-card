# Tracker Score 알고리즘 검증 결과

## 1. 검증 목적

티어 기반 스코어링 알고리즘(1~99)이 다양한 랭크 구간에서 자연스러운 점수를 산출하는지 확인한다.

---

## 2. 검증 데이터 출처

- tracker.gg 시즌 V26: A1 경쟁전 통계
- 수집 방법: Playwright MCP를 통한 자동 수집 + 수동 입력
- 수집 일시: 2026-03-18

---

## 3. 알고리즘 요약

```
Score = round(base + performance × (ceiling - base))

performance = ACS×0.25 + K/D×0.20 + ADR×0.15 + KAST×0.15 + Win×0.15 + HS%×0.10
```

정규화 기준: K/D÷2.0, ACS÷350, HS%÷35, ADR÷200, Win÷100, (KAST-40)÷50

## 6. 분석

### 정상 작동 확인

- 티어 순서대로 점수가 자연스럽게 증가
- 같은 티어 내에서 스탯 차이에 따라 합리적으로 분화
  - 골드 2(43) < 골드 3(44): 디비전 차이 반영
  - 불멸 1(89) < 불멸 1(91): 최근 전적 차이 반영

### 오버랩 작동 확인

- 실버 3(33) vs 골드 하한(34): 1점 차이로 인접 오버랩 경계에 위치
- 불멸 1(89) vs 레디언트 하한(89): 정확히 오버랩 구간에서 겹침

### 미검증 구간

- 아이언, 브론즈, 플래티넘, 초월자 티어 데이터 미수집
- 향후 추가 검증 필요

## 7. 결론 (v1)

알고리즘이 실버~레디언트 구간(7개 티어)에서 **자연스럽고 일관된 점수**를 산출함을 확인.
단, 3가지 문제점 발견 → Phase 1 개선 적용.

---

## 8. Phase 1 개선 후 재검증

### 8.1 개선 내용

| 항목 | 기존 | 개선 |
|------|------|------|
| 정규화 | 선형 `value / max` | 제곱근 `√(value / max)` (K/D, ACS, HS%, ADR) |
| KAST 범위 | 40%~90% | 65%~90% (민감도 향상) |
| 승률 | 단순 평균 | 베이지안 `(wins+5)/(total+10)` |
| 디비전 | 미반영 | `adjustedBase = base + divOffset × range × 0.3` |
| 가중치 | ACS 25, K/D 20, ADR 15, KAST 15, Win 15, HS% 10 | ACS 30, K/D 25, ADR 15, HS% 15, Win 10, KAST 5 |

### 8.3 개선 효과 분석

**Performance 분포 확장**
- v1: 0.56 ~ 0.79 (범위 0.23)
- v2: 0.69 ~ 0.83 (범위 0.14이지만 상한이 높아져 점수 활용 구간이 넓어짐)

**디비전 반영 작동**
- 골드 2(46) < 골드 3(47): 디비전에 의한 adjustedBase 차이 반영
- 실버 3(36): 디비전 3이므로 base 23 → adjustedBase 27.8

**승률 보정 작동**
- Player 2 (90% 승률): 베이지안 보정 → 76.7%로 완화, 점수 97 유지 (과도한 상승 없음)

**KAST 가중치 축소**
- KAST 15% → 5%: 차별력 없는 지표의 영향 최소화

### 8.4 결론 (v2)

Phase 1 개선으로 디비전 차이, 승률 극단값, KAST 차별력 문제가 해결됨.
알고리즘이 실버~레디언트 구간에서 안정적으로 작동함을 확인.
